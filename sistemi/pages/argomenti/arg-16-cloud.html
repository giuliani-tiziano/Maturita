Cloud, Virtualizzazione
e Active Directory

L'infrastruttura IT moderna: dal datacenter al cloud.

L'infrastruttura IT moderna si basa su cloud computing, virtualizzazione e gestione centralizzata delle identità (Active Directory). Questi tre pilastri permettono scalabilità, efficienza e sicurezza.

«Il cloud non è il futuro — è il presente.»— Gartner, 2023

02

Virtualizzazione

La virtualizzazione permette di eseguire più sistemi operativi su un singolo hardware fisico tramite un hypervisor (VMM — Virtual Machine Monitor).

TipoFunzionamentoEsempiUso
Type 1 (Bare Metal)Direttamente sull'hardware fisico — massime prestazioniVMware ESXi, Microsoft Hyper-V, ProxmoxDatacenter enterprise
Type 2 (Hosted)Sopra un OS host (Windows/Linux)VirtualBox, VMware WorkstationDesktop, sviluppo, testing

Vantaggi della virtualizzazione

  • Consolidamento server (meno HW fisico)
  • Risparmio energetico
  • Isolamento dei sistemi
  • Snapshot e backup rapidi
  • Live migration senza downtime
  • Provisioning rapido (VM in minuti)
03

Disaster Recovery e Backup

RTO e RPO

  • RTO (Recovery Time Objective): tempo massimo per ripristinare il sistema. Es. RTO=4h: il sistema deve essere operativo entro 4 ore dal guasto.
  • RPO (Recovery Point Objective): quantità massima di dati persi accettabile. Es. RPO=1h: backup ogni ora — si perderà al massimo 1 ora di dati.

Regola 3-2-1

3 copie dei dati · 2 supporti diversi (es. disco + nastro) · 1 copia fuori sede (cloud o sede secondaria). Protegge da: guasto hardware, disastri fisici, ransomware.

Clustering HA ≠ Backup!

Il clustering (Active-Passive, Active-Active) garantisce alta disponibilità: se un nodo cade, il servizio continua. Ma tutti i nodi accedono agli stessi dati — un ransomware li cifra su tutti i nodi. Il backup è separato e indispensabile.

04

Active Directory

Active Directory è il servizio Microsoft per la gestione centralizzata di utenti, computer, gruppi e policy di sicurezza in un'organizzazione.

Componenti principali

  • Domain Controller (DC): server con il database AD. Gestisce autenticazione (Kerberos/NTLM) e autorizzazione.
  • Domain: raggruppamento logico con un'unica policy di sicurezza.
  • OU (Organizational Unit): contenitore per organizzare utenti/computer (es. per reparto).
  • GPO (Group Policy Object): regole applicate automaticamente — blocco USB, screensaver, Windows Update, policy password.
  • Forest/Tree: struttura gerarchica di più domini.
  • SSO (Single Sign-On): un login per accedere a tutte le risorse autorizzate.
  • LDAP: protocollo per interrogare la directory (porta 389/636).
  • Kerberos: protocollo di autenticazione interno ad AD.

Azure AD

Versione cloud-native di Active Directory. Usa OAuth 2.0 e SAML invece di Kerberos. Gestisce accesso a applicazioni cloud e smart working. Non è una replica di AD on-premise — è un servizio IAM cloud.
