VLAN
Virtual Local Area Network

Segmentazione logica della rete su infrastruttura fisica condivisa.

Una VLAN è un dominio di broadcast logico creato sullo switch. Permette di segmentare una rete fisica in più reti logiche isolate, indipendentemente dalla topologia fisica.

Vantaggi VLAN

  • Sicurezza: isolamento del traffico tra reparti
  • Performance: riduzione del dominio di broadcast
  • Flessibilità: organizzazione logica indipendente dalla fisica
  • Gestione: policy diverse per ogni VLAN

Esempio pratico

Un'azienda con Uffici, Server e VoIP sullo stesso switch fisico. Tre VLAN separate: VLAN 10 (Uffici), VLAN 20 (Server), VLAN 30 (VoIP). Il traffico di ogni VLAN è isolato dagli altri.

01

802.1Q Trunking

Per trasportare traffico di più VLAN su un singolo cavo tra switch (o switch-router), si usa il trunk.

Access vs Trunk Port

TipoFunzioneTag 802.1QUso
Access portAppartiene a UNA sola VLANRimuove il tag (traffico non taggato verso host)Collegamento PC, stampanti, AP
Trunk portTrasporta PIÙ VLANAggiunge tag 802.1Q (4 byte) con VLAN IDSwitch-Switch, Switch-Router

Header 802.1Q

Il tag 802.1Q viene inserito nell'header Ethernet dopo i MAC address: 4 byte con TPID (0x8100) + TCI (PCP 3 bit + DEI 1 bit + VLAN ID 12 bit). VLAN ID: 1-4094. La VLAN nativa (default VLAN 1) passa senza tag.

02

Inter-VLAN Routing

Gli host in VLAN diverse NON possono comunicare direttamente — servono indirizzi IP diversi e un router. Due approcci:

Router-on-a-Stick

Un solo collegamento fisico switch-router configurato come trunk. Il router crea subinterface logiche (una per VLAN) con l'IP gateway di ogni VLAN. Cisco: interface fa0/0.10 + encapsulation dot1Q 10.

Layer 3 Switch (MLS)

Uno switch multilayer (Layer 3) può fare routing direttamente tra VLAN con Switch Virtual Interface (SVI): interface vlan 10 + ip address 192.168.10.1 255.255.255.0. Più efficiente del router-on-a-stick.

03

Configurazione Cisco IOS

! Creare le VLAN
vlan 10
  name UFFICI
vlan 20
  name SERVER

! Porta access (host)
interface fa0/1
  switchport mode access
  switchport access vlan 10

! Porta trunk (verso altro switch/router)
interface gi0/1
  switchport mode trunk
  switchport trunk allowed vlan 10,20

! Verifica
show vlan brief
show interfaces trunk
