Sistemi e Reti · Argomento 2

Subnetting,
VLSM e CIDR

Dividere le reti in sottoreti per un uso ottimale degli indirizzi IP.

Il subnetting è la tecnica che permette di dividere una rete IP in sottoreti più piccole. È essenziale per la progettazione di reti moderne, il risparmio di indirizzi IP e la separazione logica delle reti.

Competenze acquisite

  • Calcolare indirizzo di rete, broadcast, primo e ultimo host
  • Applicare la formula 2^n - 2 per il numero di host
  • Progettare una suddivisione VLSM ottimizzata
  • Convertire tra notazione decimale e CIDR
  • Calcolare la wildcard mask
«Il subnetting è l'arte di dare a ogni rete esattamente lo spazio che serve — né di più, né di meno.» — Principio VLSM
01

Subnetting

Il subnetting prende una rete IP e la suddivide in reti più piccole (sottoreti) usando una subnet mask più lunga rispetto alla classe originale. Ogni sottorete ha il proprio indirizzo di rete, broadcast e range di host.

Perché fare subnetting?

  • Separazione logica dei reparti (VLAN / sottoreti distinte)
  • Riduzione dei domini di broadcast
  • Uso efficiente degli indirizzi IP
  • Sicurezza: isolamento di segmenti di rete

02

Calcolo Subnetting

Formula fondamentale

Host disponibili = 2^n - 2

Dove n = numero di bit nella parte host. Si sottraggono 2 per: indirizzo di rete (tutti i bit host a 0) e indirizzo broadcast (tutti i bit host a 1).

Esempio: 172.16.5.0/25

  • Maschera: /25 = 255.255.255.128
  • Bit host: 32 - 25 = 7
  • Indirizzo di rete: 172.16.5.0 (bit host tutti 0)
  • Broadcast: 172.16.5.127 (bit host tutti 1 → 2^7-1 = 127)
  • Primo host: 172.16.5.1
  • Ultimo host: 172.16.5.126
  • Host disponibili: 2^7 - 2 = 126

Procedura passo a passo

  1. Individua il numero di bit host: n = 32 - prefisso CIDR
  2. Calcola i blocchi della subnet: il "salto" tra reti = 256 - ultimo ottetto della maschera
  3. Indirizzo di rete = blocco di partenza
  4. Broadcast = prossimo indirizzo di rete - 1
  5. Primo host = indirizzo rete + 1; Ultimo host = broadcast - 1
03

VLSM — Variable Length Subnet Mask

Il VLSM permette di usare maschere di lunghezza diversa per sottoreti diverse all'interno dello stesso spazio di indirizzamento. È possibile solo con protocolli classless (RIPv2, OSPF, EIGRP).

Esempio VLSM: rete 192.168.10.0/24

Requisiti: Sede A 60 host, Sede B 30 host, Sede C 10 host, Link WAN 2 host.

Strategia VLSM: ordina dal più grande al più piccolo, assegna le reti in sequenza senza sovrapporre.

04

CIDR e Wildcard Mask

CIDR (Classless Inter-Domain Routing) supera il sistema delle classi fisse permettendo blocchi di qualsiasi dimensione. Usa la notazione /n.

Wildcard Mask

È il complemento della subnet mask: Wildcard = 255.255.255.255 - subnet mask

I bit a 0 nella wildcard significano "controlla questo bit" (deve corrispondere). I bit a 1 significano "ignora questo bit".

  • /24 → SM 255.255.255.0 → Wildcard 0.0.0.255
  • /16 → SM 255.255.0.0 → Wildcard 0.0.255.255
  • host 192.168.1.5 = 192.168.1.5 0.0.0.0
  • any = 0.0.0.0 255.255.255.255

La wildcard mask si usa nelle ACL Cisco e in OSPF per specificare le reti da annunciare.
