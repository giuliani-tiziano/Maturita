Sistemi e Reti · Argomento 14

ACL
Access Control List

Il meccanismo Cisco per filtrare il traffico sulle interfacce del router.

Le ACL sono liste di regole (ACE) applicate alle interfacce di router (o switch L3) per controllare il traffico in ingresso (inbound) o uscita (outbound).

3 Utilizzi principali

  • Sicurezza: limitare quali host/reti possono accedere a determinati servizi.
  • Ottimizzazione traffico: impedire routing di certi tipi di traffico.
  • Filtro per tipo di servizio: bloccare certi protocolli o porte.

Funzionamento

Lettura top-down: la prima regola (ACE) che corrisponde al pacchetto viene applicata (permit o deny) e il processing si ferma. Se nessuna regola corrisponde → deny implicito (tutto bloccato).

⚠ Perché la regola di posizionamento?

  • ACL Standard vicino alla destinazione: filtra solo sorgente. Se messa vicino alla sorgente, bloccherebbe il traffico verso TUTTE le destinazioni, non solo quella voluta.
  • ACL Estesa vicino alla sorgente: permette di bloccare il traffico il prima possibile, risparmiando risorse di rete.
02

Wildcard Mask

La wildcard mask nelle ACL è l'inverso della subnet mask. Bit 0 = controlla (il bit dell'indirizzo deve corrispondere), bit 1 = ignora.

Calcolo

Wildcard = 255.255.255.255 - subnet mask

  • /24 → SM 255.255.255.0 → WC 0.0.0.255 (ultimi 8 bit liberi)
  • /16 → SM 255.255.0.0 → WC 0.0.255.255
  • /8 → SM 255.0.0.0 → WC 0.255.255.255

Parole chiave speciali

  • any = 0.0.0.0 255.255.255.255 — qualsiasi indirizzo IP.
  • host 192.168.1.5 = 192.168.1.5 0.0.0.0 — esattamente quell'IP.
03

Sintassi Cisco IOS

ACL Standard

access-list 10 deny host 192.168.1.100
access-list 10 permit any
!
! Applicazione:
interface fastEthernet 0/1
 ip access-group 10 out

ACL Estesa

! Blocca Telnet (23) dalla rete 10.0.0.0/8:
access-list 110 deny tcp 10.0.0.0 0.255.255.255 any eq 23
access-list 110 permit ip any any
!
interface serial 0/0
 ip access-group 110 in

Comandi di verifica

show ip access-lists                    ! Mostra ACL con contatori match
show running-config | section access-list  ! Mostra ACL nella configurazione
04

Posizionamento e Direzione

Inbound vs Outbound

DirezioneQuando viene applicataEfficienza
InboundIl pacchetto viene filtrato PRIMA del processo di routingPiù efficiente — il router non spreca risorse sul routing se il pacchetto è bloccato
OutboundIl pacchetto viene filtrato DOPO il processo di routingMeno efficiente — il routing avviene anche per pacchetti poi bloccati

⚠ Regole critiche

  • Le regole più specifiche vanno PRIMA di quelle generali.
  • Non dimenticare il deny implicito — aggiungi sempre un permit any se serve.
  • Una sola ACL per direzione per interfaccia.
