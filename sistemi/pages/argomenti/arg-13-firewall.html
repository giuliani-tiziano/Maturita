Sistemi e Reti · Argomento 13

Firewall e
Architetture di Rete

La prima linea di difesa della rete perimetrale.

Un firewall è un dispositivo (hardware o software) che controlla il traffico di rete e decide cosa permettere o bloccare secondo una security policy.

Zone di sicurezza

  • Internet: rete esterna non fidata.
  • DMZ (Demilitarized Zone): zona intermedia — server pubblici accessibili ma isolati dalla LAN.
  • LAN interna: rete protetta, risorse riservate agli utenti fidati.

Policy di base

  • Default-Deny: tutto vietato tranne il permesso esplicitamente. Più sicuro. Principio del minimo privilegio.
  • Default-Allow: tutto permesso tranne il vietato. Sconsigliato.
«Un firewall non è sufficiente. È il primo strato di difesa, non l'unico.»— SANS Institute, Defense in Depth

SPI — State Table

Il firewall SPI mantiene una state table con tutte le connessioni attive: IP/porta sorgente e destinazione, protocollo, stato TCP (SYN/ESTABLISHED/FIN), sequence number. Un pacchetto che non corrisponde a nessuna connessione aperta viene bloccato automaticamente.

02

Architetture di Firewall

DMZ — Demilitarized Zone

La DMZ ospita i server pubblici (web, mail, DNS, FTP). Questi server sono raggiungibili da Internet ma sono isolati dalla LAN interna. Se un server DMZ viene compromesso, l'attaccante non ha accesso diretto alla LAN.

Screened Subnet (la più sicura)

Internet → Firewall esterno → DMZ → Firewall interno → LAN

Due firewall separati: il FW esterno protegge la DMZ da Internet; il FW interno protegge la LAN dalla DMZ. Anche se la DMZ è completamente compromessa, il firewall interno protegge la LAN.

Bastion Host

Server esposto rinforzato: SO hardened (servizi minimi, aggiornamenti continuativi), MFA obbligatoria, logging completo. Funziona come jump host controllato per l'accesso SSH alla rete interna.

03

Limiti del Firewall

  • Non filtra il traffico che non lo attraversa (attacchi interni alla LAN).
  • Non protegge da insider threat (dipendenti malintenzionati).
  • Non ispeziona traffico cifrato senza SSL inspection.
  • Configurazione errata = falso senso di sicurezza.
  • Non rileva malware nel traffico applicativo legittimo (senza DPI).

Strategie complementari

  • IDS/IPS per rilevamento intrusioni
  • Antivirus/EDR sugli endpoint
  • Segmentazione della rete (VLAN)
  • VPN per accesso remoto sicuro
  • MFA per autenticazione forte
  • WAF per proteggere le web app
