Sistemi e Reti · Argomento 3

NAT, PAT
e ICMP

La traduzione degli indirizzi e il protocollo di diagnostica di rete.

NAT (Network Address Translation) permette a dispositivi con IP privati di comunicare su Internet traducendo gli indirizzi. ICMP è il protocollo di diagnostica usato da ping e traceroute.

Competenze acquisite

  • Distinguere NAT statico, dinamico e PAT

«NAT ha salvato IPv4 dall'esaurimento, ma a un costo architetturale.»— IETF RFC 3022
01

NAT — Network Address Translation

NAT traduce gli indirizzi IP privati in indirizzi pubblici quando i pacchetti attraversano il router verso Internet.

TipoFunzionamentoUso tipico
NAT StaticoMappatura 1:1 fissa: un IP privato ↔ un IP pubblicoServer interni accessibili dall'esterno
NAT DinamicoPool di IP pubblici assegnati dinamicamente alle sessioniPiù host con pool limitato di IP pubblici
PAT / NAT OverloadMolti IP privati condividono un solo IP pubblico via porte diverseRouter domestici, quasi tutte le LAN

Vantaggi e Svantaggi NAT

  • Pro: risparmio indirizzi IPv4; sicurezza (IP interni nascosti); indipendenza dall'ISP
  • Contro: rompe il modello end-to-end; problemi con SIP/FTP; complicazioni per server interni; latenza aggiuntiva
02

PAT — Port Address Translation

PAT (anche chiamato NAT Overload) è il tipo più diffuso di NAT. Molti IP privati condividono un singolo IP pubblico, differenziati dalla porta TCP/UDP.

Funzionamento PAT

Il router mantiene una tabella NAT con le sessioni attive:

IP PrivatoPorta priv.IP PubblicoPorta pub.Protocollo
192.168.1.51234580.100.1.150001TCP
192.168.1.73210080.100.1.150002TCP
192.168.1.84567880.100.1.150003UDP

Quando arriva la risposta da Internet all'IP pubblico, il router consulta la tabella e la smista al client corretto in base alla porta pubblica.

Comando Cisco IOS per NAT/PAT

ip nat inside source list 1 interface fa0/0 overload
!
interface fa0/0
 ip nat outside
interface fa0/1
 ip nat inside
03

ICMP — Internet Control Message Protocol

ICMP è un protocollo di supporto a IP che trasmette messaggi di controllo ed errore. Non trasporta dati utente ma è fondamentale per la diagnostica di rete.

Tipo ICMPDescrizioneUso
Echo Request / ReplyRichiesta/risposta ecoComando ping
Destination UnreachableDestinazione non raggiungibileHost/rete/porta non disponibile
Time ExceededTTL scaduto a 0Usato da traceroute
RedirectSuggerisce un percorso miglioreOttimizzazione routing

TTL e Traceroute

Il campo TTL (Time To Live) dell'header IP viene decrementato di 1 da ogni router. Quando arriva a 0, il pacchetto viene scartato e il router invia un messaggio ICMP Time Exceeded all'origine.

Traceroute sfrutta questo meccanismo: invia pacchetti con TTL=1, poi 2, poi 3… Ogni router che scarta il pacchetto rivela il proprio IP tramite ICMP Time Exceeded. Così si ricostruisce il percorso hop per hop.
