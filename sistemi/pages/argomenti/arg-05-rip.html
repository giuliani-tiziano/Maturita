RIP
Routing Information Protocol

Il protocollo Distance Vector più semplice per reti di piccole dimensioni.

RIP è un protocollo IGP di tipo Distance Vector basato sull'algoritmo di Bellman-Ford. Usa l'hop count come metrica: il numero di router attraversati per raggiungere la destinazione. È adatto a reti piccole e medie (max 15 hop).

Caratteristiche principali

  • Metrica: hop count (max 15 · 16 = irraggiungibile)
  • Aggiornamenti periodici ogni 30 secondi
  • Tipo: Distance Vector (Bellman-Ford)
  • Classificazione: IGP
  • Adatto a reti piccole/medie
«RIP è l'introduzione al routing dinamico: semplice da capire, limitato in scala.»— Cisco Networking Academy
01

Funzionamento RIP

Processo passo a passo

  1. All'avvio, ogni router conosce solo le reti direttamente connesse (metrica 1).
  2. Ogni 30 secondi, invia un RIP Update ai vicini con la propria tabella di routing.
  3. Il router ricevente aggiunge 1 alla metrica di ogni rotta ricevuta.
  4. Aggiorna la propria tabella se trova un percorso con metrica inferiore.
  5. Dopo alcuni cicli, tutti i router convergono con informazioni complete sulla rete.

02

Count-to-Infinity e Soluzioni

Il Count-to-Infinity è il problema principale di RIP: quando un link cade, i router continuano ad annunciare la rotta con metrica crescente (1→2→3…→16) finché non raggiungono il valore "infinito" (16).

Soluzioni

  • Split Horizon: un router non ri-annuncia una rotta verso il router da cui l'ha appresa. Se R1 ha imparato "rete X via R2", R1 non annuncia "rete X" a R2.
  • Poisoned Reverse: variante di Split Horizon: invece di omettere la rotta, la ri-annuncia con metrica 16 verso il router di origine. Più esplicito e veloce.
  • Triggered Update: al cambiamento di topologia, invia immediatamente un aggiornamento senza aspettare i 30 secondi periodici. Accelera notevolmente la convergenza.

04

Comandi Cisco IOS per RIP

Configurazione RIPv2

Router(config)# router rip
Router(config-router)# version 2
Router(config-router)# network 192.168.1.0
Router(config-router)# network 10.0.0.0
Router(config-router)# no auto-summary

Verifica RIP

show ip protocols     ! Protocolli attivi, reti, timer
show ip route         ! Tabella routing (R = rotte RIP)
show ip rip database  ! Database RIP con metriche e next-hop
debug ip rip          ! Messaggi RIP in tempo reale (disab. con: no debug ip rip)
