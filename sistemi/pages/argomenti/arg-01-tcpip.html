Il fondamento di qualsiasi comunicazione su Internet.

Di cosa si tratta

Il modello TCP/IP è l'architettura di protocolli che governa la comunicazione su Internet. Ogni dispositivo connesso usa questi protocolli per inviare e ricevere dati.

L'indirizzamento IP assegna un identificatore univoco a ogni interfaccia di rete. Capire le classi IP, gli indirizzi privati e la subnet mask è fondamentale per qualsiasi lavoro di rete.

01

Il Modello TCP/IP

Il modello TCP/IP (Transmission Control Protocol / Internet Protocol) è un modello a 4 livelli che descrive come i dati vengono trasmessi su Internet. È il modello pratico su cui Internet è costruita.

Livello TCP/IPCorrispondenza OSIProtocolli principaliFunzione
4. ApplicazioneLivelli 5-6-7 OSIHTTP, HTTPS, FTP, DNS, SMTP, SSHInterfaccia con le applicazioni
3. TrasportoLivello 4 OSITCP, UDPComunicazione end-to-end, porte
2. InternetLivello 3 OSIIP, ICMP, ARPIndirizzamento logico, routing
1. Accesso alla reteLivelli 1-2 OSIEthernet, Wi-Fi, PPPTrasmissione fisica, MAC address

Incapsulamento

Ogni livello aggiunge il proprio header ai dati: dati applicazione → segmento TCP (+ header TCP) → pacchetto IP (+ header IP) → frame Ethernet (+ header/trailer). Il processo inverso alla ricezione si chiama de-incapsulamento.

02

Indirizzi IP

Un indirizzo IPv4 è un numero a 32 bit scritto in notazione decimale puntata: quattro ottetti separati da punti (es. 192.168.1.100). Ogni indirizzo identifica un'interfaccia di rete.

Indirizzi privati (RFC 1918)

Non instradabili su Internet, usati nelle reti locali con NAT:

Indirizzi speciali

  • Loopback: 127.0.0.1 (qualsiasi 127.x.x.x) — pacchetti tornano allo stesso host. Usato per test locali.
  • Indirizzo di rete: bit host tutti a 0 (es. 192.168.1.0/24). Non assegnabile agli host.
  • Broadcast: bit host tutti a 1 (es. 192.168.1.255/24). Raggiungerebbe tutti gli host della rete.
  • 0.0.0.0: "qualsiasi interfaccia" — usato in configurazioni di routing.
  • 255.255.255.255: broadcast limitato — tutti gli host della rete locale.

Subnet Mask e CIDR

La subnet mask separa la parte rete dalla parte host. I bit a 1 identificano la rete, i bit a 0 identificano l'host.

La notazione CIDR (Classless Inter-Domain Routing) usa il formato /n dove n è il numero di bit della parte rete: es. 192.168.1.0/24 = subnet mask 255.255.255.0.

Three-Way Handshake TCP

Prima di trasmettere dati, TCP stabilisce una connessione in 3 passi:

  1. SYN: il client invia un segmento SYN al server per iniziare la connessione.
  2. SYN-ACK: il server risponde con SYN-ACK (sincronizzazione + conferma).
  3. ACK: il client invia ACK per confermare la ricezione del SYN-ACK.

Porte

Le porte sono numeri a 16 bit (0–65535) che identificano uno specifico processo o servizio su un host. Permettono il multiplexing: più comunicazioni su un singolo indirizzo IP.

05

IPv6

IPv4 ha circa 4,3 miliardi di indirizzi (2³²), ormai insufficienti per il numero di dispositivi connessi. IPv6 usa 128 bit → 3,4 × 10³⁸ indirizzi — essenzialmente inesauribili.

CaratteristicaIPv4IPv6
Lunghezza indirizzo32 bit128 bit
FormatoDecimale puntato (es. 192.168.1.1)Esadecimale con : (es. 2001:db8::1)
Indirizzi disponibili~4,3 miliardi3,4 × 10³⁸
ConfigurazioneDHCP o manualeSLAAC (autoconfigurazione) o DHCPv6
HeaderVariabile (20-60 byte)Fisso (40 byte) + extension headers
NATNecessarioNon necessario
SicurezzaIPsec opzionaleIPsec integrato (nativo)

Formato indirizzo IPv6

8 gruppi da 4 cifre esadecimali separate da :

2001:0db8:85a3:0000:0000:8a2e:0370:7334

Abbreviazioni: gruppi di tutti 0 → :: (una sola volta nell'indirizzo). Zeri iniziali nei gruppi si omettono.
