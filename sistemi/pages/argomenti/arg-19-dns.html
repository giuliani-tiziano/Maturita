DNS
Domain Name System

Il "registro telefonico" di Internet: traduce nomi in indirizzi IP.

Il DNS traduce i nomi di dominio (es. www.google.com) in indirizzi IP (es. 142.250.180.46). È un sistema distribuito e gerarchico — nessun singolo server conosce tutti i nomi.

Perché il DNS è fondamentale

  • Gli IP cambiano, i nomi restano stabili
  • Load balancing: un nome → più IP
  • Un IP → più nomi (virtual hosting)
  • Porta UDP/TCP 53
«Il DNS è la colonna vertebrale di Internet — senza di esso, non si naviga.»— Internet Engineering Task Force
01

Gerarchia DNS

Il DNS è organizzato in livelli:

  • Root (.): 13 root server gestiti da IANA. Conoscono i server TLD.
  • TLD (Top-Level Domain): .com, .it, .org, .net — gestiti da ICANN/NIR. Conoscono i server authoritative.
  • Authoritative DNS: server che conosce i record di una zona specifica (es. google.com). Risposta definitiva.
  • DNS Ricorsivo/Resolver: il server DNS dell'ISP o di Google (8.8.8.8) che fa le query per conto del client.

03

Risoluzione DNS: Ricorsiva vs Iterativa

Risoluzione Ricorsiva (client → resolver)

Il client invia la query al resolver (es. 8.8.8.8) e ASPETTA la risposta completa. Il resolver fa tutto il lavoro.

Risoluzione Iterativa (resolver → server autoritativi)

  1. Resolver → Root: "chi sa di .com?" → Root risponde con i TLD servers
  2. Resolver → TLD .com: "chi sa di google.com?" → risponde con ns1.google.com
  3. Resolver → ns1.google.com: "qual è l'IP di www.google.com?" → risponde con 142.250.x.x
  4. Resolver → Client: "ecco l'IP + TTL"

Il risultato viene cachato per il valore del TTL (Time To Live) del record.

Sicurezza DNS

  • DNS Poisoning (Cache Poisoning): iniettare risposte false nella cache del resolver → utente reindirizzato a siti malevoli.
  • DNSSEC: firma crittografica dei record DNS per garantirne l'autenticità.
  • DNS over HTTPS (DoH): cifra le query DNS dentro HTTPS — privacy.
