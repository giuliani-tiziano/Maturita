Cos'è un algoritmo

Un algoritmo è una sequenza finita di istruzioni non ambigue che risolvono un problema. La struttura dati è l'organizzazione in memoria dei dati su cui l'algoritmo opera. La scelta di struttura e algoritmo giusti può fare la differenza tra un programma lento e uno veloce di ordini di grandezza.

Notazione Big-O

Big-O descrive il limite superiore asintotico della complessità al crescere dell'input n. Non misura tempi reali (dipendono dall'hardware) ma come cresce il tempo/spazio.

Casi migliore, medio, peggiore

  • Caso migliore (Ω, Omega): input ideale. Ricerca lineare: elemento in posizione 0 → O(1).
  • Caso medio (Θ, Theta): input casuale. Ricerca lineare: elemento a metà → O(n/2) = O(n).
  • Caso peggiore (O, Big-O): input avverso. Ricerca lineare: elemento non trovato → O(n).

La notazione Big-O si usa quasi sempre per il caso peggiore: è la garanzia di performance.

// Esempi di complessità in JavaScript

// O(1) — costante
function primoElemento(arr) {
  return arr[0]; // indipendente da arr.length
}

// O(n) — lineare
function cercaLineare(arr, target) {
  for (let x of arr) {  // scorre tutto l'array
    if (x === target) return true;
  }
  return false;
}

// O(n²) — quadratico (due cicli annidati)
function haaDuplicati(arr) {
  for (let i = 0; i < arr.length; i++)
    for (let j = i+1; j < arr.length; j++)
      if (arr[i] === arr[j]) return true;
  return false;
}
// Versione O(n) con Set:
// return arr.length !== new Set(arr).size;

Array

Sequenza di elementi in memoria contigua, accessibili per indice. L'indirizzo dell'elemento i = base + i × dimensione_tipo → accesso O(1).

Inserimento/cancellazione in mezzo: O(n) perché bisogna spostare gli elementi successivi.

// Array in PHP
$arr = [10, 20, 30, 40, 50];
echo $arr[2];  // 30 → O(1)

// Inserzione in mezzo → O(n)
array_splice($arr, 2, 0, [25]);
// [10,20,25,30,40,50]

// In JavaScript
const arr = [1, 2, 3];
arr.push(4);    // O(1) in coda
arr.unshift(0); // O(n) in testa
Stack (LIFO) e Queue (FIFO)

Stack — Last In, First Out:

// PHP Stack
$stack = [];
array_push($stack, 'A'); // push
array_push($stack, 'B');
array_push($stack, 'C');
$top = array_pop($stack); // pop → 'C'
// Usi: call stack, undo/redo, parsing

Queue — First In, First Out:

// PHP Queue
$queue = [];
array_push($queue, 'A');   // enqueue
array_push($queue, 'B');
array_push($queue, 'C');
$front = array_shift($queue); // dequeue → 'A'
// Usi: coda stampa, richieste web, BFS
Albero binario di ricerca (BST)

Ogni nodo ha al massimo 2 figli. Proprietà BST: sottoalbero sinistro < nodo < sottoalbero destro.

Ricerca: O(log n) caso medio, O(n) caso peggiore (albero degenerato). BST bilanciati (AVL, Red-Black) garantiscono O(log n) sempre.

// BST — pseudocodice ricerca
function cerca(nodo, valore):
  if nodo == null:
    return null      // non trovato

  if valore == nodo.valore:
    return nodo      // trovato!

  if valore < nodo.valore:
    return cerca(nodo.sinistra, valore)
  else:
    return cerca(nodo.destra, valore)

// Ogni livello dimezza lo spazio → O(log n)

Ordinamento

Bubble Sort — O(n²)

Confronta coppie adiacenti e le scambia se fuori ordine. Ripete n passate. Semplice da capire, inefficiente per dati grandi.

// Bubble Sort in PHP
function bubbleSort(array $arr): array {
  $n = count($arr);
  for ($i = 0; $i < $n - 1; $i++) {
    for ($j = 0; $j < $n - $i - 1; $j++) {
      if ($arr[$j] > $arr[$j + 1]) {
        // Scambia
        [$arr[$j], $arr[$j+1]] =
        [$arr[$j+1], $arr[$j]];
      }
    }
  }
  return $arr;
}

Merge Sort — O(n log n) garantito

Divide et impera: divide l'array a metà ricorsivamente finché ci sono array da 1 elemento, poi fonde (merge) le coppie mantenendo l'ordine. Stabile, sempre O(n log n), richiede O(n) spazio extra.

// Merge Sort in JavaScript
function mergeSort(arr) {
  if (arr.length <= 1) return arr;

  const mid = Math.floor(arr.length / 2);
  const left  = mergeSort(arr.slice(0, mid));
  const right = mergeSort(arr.slice(mid));

  return merge(left, right);
}

function merge(left, right) {
  const result = [];
  let i = 0, j = 0;
  while (i < left.length && j < right.length) {
    if (left[i] <= right[j])
      result.push(left[i++]);
    else
      result.push(right[j++]);
  }
  return [...result, ...left.slice(i),
                     ...right.slice(j)];
}

Ricerca Binaria — O(log n)

Funziona solo su array ordinati. Ad ogni passo dimezza lo spazio di ricerca: confronta con il centro, va a destra o sinistra.

// Ricerca binaria in PHP
function ricercaBinaria(array $arr, $target): int {
  $sx = 0;
  $dx = count($arr) - 1;

  while ($sx <= $dx) {
    $mid = intdiv($sx + $dx, 2);

    if ($arr[$mid] === $target)
      return $mid;          // trovato!
    elseif ($arr[$mid] < $target)
      $sx = $mid + 1;       // cerca a destra
    else
      $dx = $mid - 1;       // cerca a sinistra
  }
  return -1; // non trovato
}
// Per 1.000.000 elementi: max 20 confronti!

Prerequisito fondamentale

L'array DEVE essere ordinato. Se non lo è, ordinarlo prima costa O(n log n) — perde senso per una singola ricerca. Utile quando si fanno molte ricerche sullo stesso array già ordinato.

Ricerca lineare vs binaria

Lineare O(n): funziona su qualsiasi array, ma lenta per n grandi. Binaria O(log n): velocissima, ma richiede ordinamento. Per n=1.000.000: lineare fino a 1.000.000 confronti, binaria solo 20.

Tabelle Hash

Una funzione hash mappa una chiave a un indice (bucket) in un array. Permette inserimento, ricerca e cancellazione in O(1) nel caso medio.

Collisione: due chiavi diverse producono lo stesso hash. Strategie: chaining (lista per bucket) o open addressing (cerca il prossimo slot libero).

// Hash table in PHP: già built-in come array!
// Gli array PHP sono hash map.

$tabella = [];
$tabella['mario'] = 25;    // insert O(1)
$eta = $tabella['mario'];  // lookup O(1)
unset($tabella['mario']);   // delete O(1)
isset($tabella['mario']);   // exist O(1)

// In JavaScript: Map o oggetto {}
const map = new Map();
map.set('mario', 25);
map.get('mario'); // 25
map.has('luigi'); // false
