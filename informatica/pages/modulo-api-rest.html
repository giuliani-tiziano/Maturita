Come progettare, consumare e proteggere API web RESTful.

Cos'è una API

Un'API (Application Programming Interface) è un contratto che definisce come due applicazioni si scambiano dati e funzionalità. Una REST API usa HTTP come protocollo e JSON come formato dati.

Esempio: quando un'app meteo mostra le previsioni, chiama un'API del servizio meteorologico che risponde con dati JSON.

Modulo 9 · Architettura

Principi
REST

I 6 vincoli REST (Fielding, 2000)
1
Stateless
Ogni richiesta contiene TUTTE le informazioni necessarie. Il server non mantiene stato del client. Il token di autenticazione va in ogni richiesta.
2
Client-Server
Separazione delle responsabilità: il client gestisce la UI, il server i dati. Possono evolvere indipendentemente.
3
Uniform Interface
URL identifica la risorsa, metodo HTTP l'azione. Interfaccia standardizzata per tutti i client.
4
Cacheable
Le risposte devono indicare se sono cacheable. Migliora le performance evitando richieste inutili.
5
Layered System
Il client non sa se comunica col server finale o con proxy/load balancer intermedi.
6
Code on Demand (opzionale)
Il server può inviare codice eseguibile al client (es. JavaScript). L'unico vincolo opzionale.
Design URL — best practice
// Risorse al plurale, lowercase, con trattino
GET    /api/utenti          → lista utenti
POST   /api/utenti          → crea utente
GET    /api/utenti/42       → dettaglio utente 42
PUT    /api/utenti/42       → aggiorna utente 42
PATCH  /api/utenti/42       → aggiorna parzialmente
DELETE /api/utenti/42       → elimina utente 42

// Risorse annidate:
GET    /api/utenti/42/ordini  → ordini dell'utente 42
POST   /api/utenti/42/ordini  → crea ordine per utente 42

// Filtri via query params (non nell'URL):
GET    /api/utenti?ruolo=admin&pagina=2&limit=10

// MAI usare verbi nell'URL:
// SBAGLIATO: /api/getUtente, /api/creaOrdine
// GIUSTO:    GET /api/utenti,  POST /api/ordini

Status Codes principali

2xx — Successo

  • 200 OK — richiesta riuscita
  • 201 Created — risorsa creata (POST)
  • 204 No Content — ok, nessun corpo (DELETE)

4xx — Errore client

  • 400 Bad Request — input non valido
  • 401 Unauthorized — non autenticato
  • 403 Forbidden — non autorizzato
  • 404 Not Found — risorsa non esiste
  • 429 Too Many Requests

5xx — Errore server

  • 500 Internal Server Error
  • 503 Service Unavailable
Modulo 9 · Sicurezza

Autenticazione
nelle API

API Key

Stringa segreta assegnata ad ogni client. Semplice da implementare, adatta per API machine-to-machine.

// Client: invia l'API key nell'header
// (non nell'URL — verrebbe logata)
GET /api/dati
X-API-Key: abc123secretkey456

// Server PHP: verifica
$key = $_SERVER['HTTP_X_API_KEY'] ?? '';
if (!validateApiKey($key)) {
    http_response_code(401);
    echo json_encode(['error' => 'Unauthorized']);
    exit;
}
JWT — JSON Web Token

Token firmato digitalmente. Struttura: Header.Payload.Signature (Base64URL). Il server verifica la firma senza consultare il DB → stateless.

// Struttura JWT decodificata:
// Header:
{"alg": "HS256", "typ": "JWT"}

// Payload (claim):
{
  "sub": "42",           // user ID
  "nome": "Mario Rossi",
  "ruolo": "admin",
  "iat": 1700000000,     // issued at
  "exp": 1700086400      // expiration
}

// Il client invia nelle richieste:
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...

// Server PHP legge:
$authHeader = $_SERVER['HTTP_AUTHORIZATION'];
$token = str_replace('Bearer ', '', $authHeader);
// Verifica firma con libreria JWT
Modulo 9 · Client

Consumare
API esterne

PHP — cURL
// GET request con cURL
function apiGet(string $url, string $token = ''): array {
    $ch = curl_init();
    curl_setopt_array($ch, [
        CURLOPT_URL            => $url,
        CURLOPT_RETURNTRANSFER => true,
        CURLOPT_TIMEOUT        => 10,
        CURLOPT_HTTPHEADER     => [
            'Accept: application/json',
            'Authorization: Bearer ' . $token,
        ],
        CURLOPT_SSL_VERIFYPEER => true,
    ]);
    $response = curl_exec($ch);
    $httpCode = curl_getinfo($ch, CURLINFO_HTTP_CODE);
    curl_close($ch);

    if ($httpCode !== 200) {
        throw new Exception("API error: $httpCode");
    }
    return json_decode($response, true);
}

// POST request
curl_setopt($ch, CURLOPT_POST, true);
curl_setopt($ch, CURLOPT_POSTFIELDS,
    json_encode(['nome' => 'Mario']));
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    'Content-Type: application/json'
]);
JavaScript — fetch()
// GET con fetch (Promise)
fetch('/api/utenti/42', {
  headers: {
    'Authorization': 'Bearer ' + token,
    'Accept': 'application/json'
  }
})
.then(response => {
  if (!response.ok)
    throw new Error('HTTP ' + response.status);
  return response.json();
})
.then(data => aggiornaUI(data))
.catch(err => console.error(err));

// POST con fetch
fetch('/api/utenti', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer ' + token
  },
  body: JSON.stringify({
    nome: 'Mario',
    email: 'mario@example.com'
  })
})
.then(r => r.json())
.then(nuovoUtente => console.log(nuovoUtente));
Modulo 9 · Server

Costruire una
REST API in PHP

// api.php — API REST semplice per la risorsa "utenti"
header('Content-Type: application/json');
header('Access-Control-Allow-Origin: *'); // CORS

$method = $_SERVER['REQUEST_METHOD'];
// URL: /api.php?risorsa=utenti&id=42
$risorsa = $_GET['risorsa'] ?? '';
$id      = isset($_GET['id']) ? intval($_GET['id']) : null;

// Connessione DB
$conn = new mysqli('localhost', 'user', 'pwd', 'db');
if ($conn->connect_error) {
    http_response_code(500);
    echo json_encode(['error' => 'DB connection failed']);
    exit;
}

if ($risorsa === 'utenti') {
    switch ($method) {

        case 'GET':
            if ($id) {
                // GET /api.php?risorsa=utenti&id=42
                $stmt = $conn->prepare(
                    "SELECT id, nome, email FROM utenti WHERE id = ?"
                );
                $stmt->bind_param('i', $id);
                $stmt->execute();
                $utente = $stmt->get_result()->fetch_assoc();
                if (!$utente) {
                    http_response_code(404);
                    echo json_encode(['error' => 'Non trovato']);
                } else {
                    echo json_encode($utente);
                }
            } else {
                // GET /api.php?risorsa=utenti → lista
                $result = $conn->query(
                    "SELECT id, nome, email FROM utenti"
                );
                echo json_encode($result->fetch_all(MYSQLI_ASSOC));
            }
            break;

        case 'POST':
            // Legge il body JSON
            $body = json_decode(file_get_contents('php://input'), true);
            if (!$body || !isset($body['nome'], $body['email'])) {
                http_response_code(400);
                echo json_encode(['error' => 'Dati mancanti']);
                break;
            }
            $stmt = $conn->prepare(
                "INSERT INTO utenti (nome, email) VALUES (?, ?)"
            );
            $stmt->bind_param('ss', $body['nome'], $body['email']);
            $stmt->execute();
            http_response_code(201); // Created
            echo json_encode(['id' => $conn->insert_id, ...$body]);
            break;

        case 'DELETE':
            if (!$id) { http_response_code(400); break; }
            $stmt = $conn->prepare(
                "DELETE FROM utenti WHERE id = ?"
            );
            $stmt->bind_param('i', $id);
            $stmt->execute();
            http_response_code(204); // No Content
            break;

        default:
            http_response_code(405); // Method Not Allowed
            echo json_encode(['error' => 'Metodo non supportato']);
    }
}
$conn->close();
