JavaScript
lato client

Il linguaggio del browser che rende le pagine web interattive e dinamiche.

Cosa fa JavaScript

JavaScript è l'unico linguaggio di programmazione che i browser eseguono nativamente. Agisce sulla pagina già caricata: modifica l'HTML, risponde agli eventi utente, comunica con il server in background (AJAX). PHP genera la pagina; JavaScript la anima.

Argomenti del modulo

  • Variabili (var/let/const), array, oggetti, tipi dinamici
  • Strutture di controllo e funzioni (incluse arrow function)
  • Gestione degli eventi (addEventListener, bubbling, preventDefault)
  • DOM — selezione, modifica, creazione, rimozione elementi
  • Creazione e modifica dinamica di contenuti HTML
  • Costruzione e gestione tabelle HTML dinamiche
  • Interazione CSS da JavaScript (classList, style)
  • XMLHttpRequest — richieste HTTP asincrone
  • Validazione form lato client
  • AJAX — aggiornamento pagina senza ricaricamento
  • JSON per scambio dati con il server
  • Strumenti di debugging (console, DevTools)
«JavaScript fa con il DOM quello che PHP fa con il database: legge, scrive, modifica, cancella.» — Analogia fondamentale

DOM e Gestione degli eventi

Il DOM è la rappresentazione ad albero della pagina HTML. JavaScript lo manipola in tempo reale.

// Selezione
const btn    = document.getElementById('mio-btn');
const titolo = document.querySelector('h1');          // primo h1
const links  = document.querySelectorAll('a.attivo'); // tutti i link .attivo

// Modifica contenuto
titolo.textContent = 'Nuovo titolo';           // solo testo (sicuro)
div.innerHTML = '<strong>Grassetto</strong>';  // HTML (attenzione XSS)

// Modifica attributi
img.setAttribute('src', 'nuova.jpg');
img.setAttribute('alt', 'Descrizione');
const src = img.getAttribute('src');

// Modifica CSS
el.style.color = 'red';
el.style.display = 'none';    // nasconde
el.classList.add('attivo');
el.classList.remove('disabilitato');
el.classList.toggle('selezionato');  // aggiunge/rimuove
el.classList.contains('aperto');     // true/false
// Crea elemento
const li = document.createElement('li');
li.textContent = 'Nuovo elemento';
li.classList.add('voce-lista');

// Aggiungi al DOM
const lista = document.getElementById('lista');
lista.appendChild(li);           // in fondo
lista.prepend(li);               // in testa
lista.insertBefore(li, altro);   // prima di "altro"

// Rimuovi elemento
li.remove();                     // metodo moderno
// lista.removeChild(li);        // vecchio modo

// Clona elemento
const copia = li.cloneNode(true); // true = clona anche figli
// Registra evento (modo corretto)
btn.addEventListener('click', function(event) {
    console.log('Cliccato!');
    event.preventDefault();   // blocca comportamento default (es. submit)
    event.stopPropagation();  // ferma il bubbling
});

// Con arrow function
btn.addEventListener('click', (e) => {
    console.log(e.target);    // elemento che ha ricevuto l'evento
});

// DOMContentLoaded — aspetta il DOM
document.addEventListener('DOMContentLoaded', () => {
    // sicuro: il DOM è pronto, CSS e immagini potrebbero non esserlo
    inizializza();
});

// window.onload — aspetta tutto (immagini, CSS…)
window.addEventListener('load', () => { ... });

Event bubbling — delegazione degli eventi

Quando si clicca su un elemento, l'evento si propaga verso i genitori. Questo permette la delegazione: invece di registrare un handler su ogni figlio, si registra uno solo sul genitore e si usa event.target per capire quale figlio è stato cliccato. Molto utile per liste create dinamicamente.

lista.addEventListener('click', (e) => {
    if (e.target.tagName === 'LI') {
        e.target.classList.toggle('selezionato');
    }
});
Sezione 04

AJAX e JSON

Aggiornamento della pagina senza ricaricamento tramite richieste HTTP asincrone.

// Richiesta GET
const xhr = new XMLHttpRequest();
xhr.open('GET', 'api.php?id=5', true); // true = asincrono

xhr.onreadystatechange = function() {
    // readyState: 0=UNSENT 1=OPENED 2=HEADERS 3=LOADING 4=DONE
    if (xhr.readyState === 4 && xhr.status === 200) {
        const dati = JSON.parse(xhr.responseText);
        aggiornaDom(dati);
    } else if (xhr.readyState === 4) {
        console.error('Errore HTTP: ' + xhr.status);
    }
};

xhr.send();  // invia la richiesta

// Richiesta POST
const xhrPost = new XMLHttpRequest();
xhrPost.open('POST', 'salva.php', true);
xhrPost.setRequestHeader('Content-Type', 'application/x-www-form-urlencoded');
xhrPost.send('nome=Mario&voto=28');

JSON (JavaScript Object Notation) è il formato standard per scambiare dati strutturati.

// JSON in JS
const obj   = { nome: "Mario", voto: 28, attivo: true };
const strJ  = JSON.stringify(obj); // → '{"nome":"Mario","voto":28,...}'
const obj2  = JSON.parse(strJ);    // → oggetto JS originale

// Flusso AJAX completo
const xhr = new XMLHttpRequest();
xhr.open('GET', 'studenti.php', true);
xhr.onreadystatechange = function() {
    if (xhr.readyState === 4 && xhr.status === 200) {
        const risposta = JSON.parse(xhr.responseText);
        // risposta è un array o oggetto JS
        risposta.studenti.forEach(s => {
            const li = document.createElement('li');
            li.textContent = s.nome + ': ' + s.voto;
            lista.appendChild(li);
        });
    }
};
xhr.send();
// PHP che risponde (studenti.php)
header('Content-Type: application/json');
$studenti = [['nome'=>'Mario','voto'=>28], ['nome'=>'Luigi','voto'=>30]];
echo json_encode(['studenti' => $studenti, 'totale' => count($studenti)]);
Sezione 05

Tabelle Dinamiche e Validazione Form

function creaTabella(dati, intestazioni, contenitore) {
    // Svuota il contenitore
    contenitore.innerHTML = '';

    const table = document.createElement('table');

    // Intestazione
    const thead = document.createElement('thead');
    const trH   = document.createElement('tr');
    intestazioni.forEach(col => {
        const th = document.createElement('th');
        th.textContent = col;
        trH.appendChild(th);
    });
    thead.appendChild(trH);
    table.appendChild(thead);

    // Corpo
    const tbody = document.createElement('tbody');
    dati.forEach(riga => {
        const tr = document.createElement('tr');
        Object.values(riga).forEach(valore => {
            const td = document.createElement('td');
            td.textContent = valore; // textContent è sicuro (no XSS)
            tr.appendChild(td);
        });
        tbody.appendChild(tr);
    });
    table.appendChild(tbody);
    contenitore.appendChild(table);
}

// Utilizzo
const dati = [
    {matricola: '001', nome: 'Mario', voto: 28},
    {matricola: '002', nome: 'Luigi', voto: 30}
];
creaTabella(dati, ['Matricola', 'Nome', 'Voto'],
    document.getElementById('tabella-studenti'));
document.getElementById('form-reg').addEventListener('submit', function(e) {
    const nome  = document.getElementById('nome').value.trim();
    const email = document.getElementById('email').value.trim();
    const pwd   = document.getElementById('password').value;
    const errori = [];

    if (nome.length < 2) errori.push("Nome troppo corto");
    if (!email.includes('@')) errori.push("Email non valida");
    if (pwd.length < 8) errori.push("Password min 8 caratteri");

    if (errori.length > 0) {
        e.preventDefault(); // blocca l'invio
        document.getElementById('errori').innerHTML =
            errori.map(e => `<p class="errore">${e}</p>`).join('');
    }
    // Se nessun errore, il form viene inviato normalmente
});

Validazione client ≠ sicurezza

La validazione JavaScript migliora l'esperienza utente (feedback immediato senza round-trip al server) ma non garantisce sicurezza: un utente malintenzionato può disabilitare JS o inviare richieste HTTP direttamente. La validazione vera e sicura si fa sempre lato server in PHP.

Console (F12 → Console):

  • console.log(valore) — stampa valore
  • console.error(msg) — errore in rosso
  • console.table(array) — tabella leggibile
  • console.time('label') / timeEnd — misura tempo

Pannello Sources — Breakpoints:

  • Clicca sul numero di riga per aggiungere un breakpoint
  • L'esecuzione si ferma lì: puoi ispezionare le variabili
  • Step Over (F10), Step Into (F11), Resume (F8)
  • Parola chiave debugger; nel codice — breakpoint programmatico

Pannello Network: vedi tutte le richieste HTTP, incluse quelle AJAX. Puoi ispezionare headers, payload e risposta JSON.
