Concetto centrale

DB
Relazionale
SQL
Struttura
Relazione

Chiavi
Superchiave → candidata → primaria. Chiave esterna per l'integrità referenziale
Architettura
ANSI/SPARC
Livello esterno, logico e fisico. Indipendenza dei dati

Algebra Relazionale
σ selezione, π proiezione, ⋈ join, ∪ ∩ − unione differenza
Connessione
Algebra → SQL
WHERE = σ, SELECT = π, JOIN = ⋈, UNION = ∪

M4 SQL
Concetto centrale
SQL
Linguaggio dichiarativo per definire e manipolare dati relazionali
DDL
DML
MySQL
Struttura
DDL
CREATE TABLE · ALTER TABLE · DROP · TRUNCATE. Vincoli e tipi di dato
Dati
DML
INSERT · SELECT · UPDATE · DELETE. Clausole WHERE, GROUP BY, HAVING
Combinazione tabelle
JOIN
INNER JOIN: solo corrispondenze. LEFT JOIN: tutte le righe sinistra + NULL
Query annidate
Subquery
Non correlata: eseguita una volta. Correlata: per ogni riga esterna
Oggetti avanzati
Viste · Trigger · Procedure
VIEW: tabella virtuale. Trigger: automatico su eventi. Stored procedure: riutilizzabile

M6 Progettazione DB
Processo
3 Fasi di Progettazione
Concettuale (E/R) → Logica (relazionale) → Fisica (SQL)
E/R
3NF
MySQL
Fase 1
Diagramma E/R
Entità (rettangolo), attributi (ellisse), associazioni (rombo), cardinalità
Fase 2
Normalizzazione
1NF: atomico. 2NF: no dep. parziali. 3NF: no dep. transitive
Traduzione
N:N → Tabella ponte
Crea tabella associativa con PK composta dalle due FK
Specializzazione
ISA (generalizzazione)
Entità figlia "è un tipo di" padre. Eredita attributi. Totale/parziale, esclusiva/sovrapposta
Problemi senza normalizzazione
Anomalie
Inserimento, cancellazione, aggiornamento. Causa: ridondanza dei dati
