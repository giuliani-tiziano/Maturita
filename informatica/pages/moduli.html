Concetti chiave del modulo

  • Sistema informatico e sistema informativo — differenza fondamentale
  • DBMS: definizione, funzioni, vantaggi rispetto ai file
  • Architettura ANSI/SPARC: livello esterno, logico, fisico
  • Struttura relazione: attributi, tuple, domini, schema, istanza
  • Grado (numero attributi) e cardinalità (numero tuple)
  • Superchiavi → chiavi candidate → chiave primaria → chiave esterna
  • Vincoli di dominio, unicità, NOT NULL, integrità referenziale
  • Regole di gestione: CASCADE, RESTRICT, SET NULL
  • Algebra relazionale: operatori insiemistici (∪ ∩ − ×) e relazionali (σ π ⋈ ρ)
  • Collegamento algebra relazionale ↔ SQL

Concetti chiave del modulo

  • Architettura client/server e ruolo di PHP nel ciclo richiesta/risposta
  • Variabili ($), costanti (define/const), tipi di dato, operatori
  • Scope locale/globale; variabili superglobali ($_GET, $_POST, $_SESSION…)

  • Funzioni personalizzate e anonime (lambda)
  • Form: GET vs POST, gestione input, htmlspecialchars() per sicurezza XSS
  • Cookie (setcookie) e sessioni (session_start, $_SESSION)
  • Autenticazione: password_hash() e password_verify()
  • MySQLi: connessione, query, prepared statements, fetch
  • JSON: json_encode() / json_decode()
  • Gestione file: fopen, fread, fwrite, file_get_contents

Concetti chiave del modulo

Concetti chiave del modulo

  • DDL: CREATE TABLE con tipi e vincoli, ALTER TABLE, DROP, TRUNCATE
  • DML: INSERT, UPDATE (con WHERE!), DELETE, SELECT
  • SELECT: WHERE, ORDER BY (ASC/DESC), GROUP BY, HAVING, LIMIT, DISTINCT

  • Funzioni MySQL: numeri, stringhe, DATE_FORMAT(), IF()

  • UNION vs UNION ALL
  • Viste (VIEW): creazione, vantaggi, limitazioni
  • Trigger: BEFORE/AFTER, FOR EACH ROW, NEW/OLD
  • Stored procedure: CALL, IN/OUT parametri
  • INSERT IGNORE e ON DUPLICATE KEY UPDATE (upsert)
  • Indici: performance, CREATE INDEX, indici su FK

Concetti chiave del modulo

  • Stack: HTML/CSS + JavaScript (client) + PHP (server) + MySQL (DB)
  • Flusso AJAX: JS → XHR → PHP → SQL → JSON → JS → DOM
  • SQL Injection: attacco + prevenzione con prepared statements
  • XSS: attacco + prevenzione con htmlspecialchars() / textContent

  • CRUD: Create/Read/Update/Delete → INSERT/SELECT/UPDATE/DELETE
  • Viste SQL in applicazioni PHP: vantaggi pratici
  • Fasi di sviluppo: analisi → progettazione → sviluppo → test → deploy

Progettazione
Concettuale, Logica e Fisica
Diagramma E/R. Entità deboli. ISA. Traduzione E/R→relazionale. Normalizzazione 1NF/2NF/3NF. Implementazione MySQL.
Quiz →

Concetti chiave del modulo

  • 3 fasi: concettuale (E/R) → logica (relazionale) → fisica (SQL)
  • E/R: entità, attributi, associazioni, cardinalità (1:1, 1:N, N:N)
  • Associazioni: binarie, unarie (ricorsive), multiple (ternarie)
  • Entità deboli: dipendenza dall'entità forte, chiave parziale
  • Generalizzazioni ISA: subset, totale/parziale, esclusiva/sovrapposta
  • Ristrutturazione E/R: attributi multivalore, generalizzazioni
  • Traduzione E/R→relazionale: regole per 1:1, 1:N, N:N, entità deboli
  • Dipendenze funzionali: A→B, determinante, parziali, transitive
  • 1NF: attributi atomici. 2NF: no dipendenze parziali. 3NF: no dipendenze transitive
  • Anomalie: inserimento, cancellazione, aggiornamento
  • Progettazione fisica: tipi di dato MySQL, vincoli SQL, indici
