Computer Networks

LAN · WAN · OSI Model · TCP/IP · Topologies · Protocols

Overview

What is a computer network?

A computer network is a set of devices (computers, servers, printers, smartphones) connected together to share resources and communicate. Networks are the foundation of the internet, cloud computing, email, streaming, and virtually every digital service. Understanding how data travels from one point to another — through protocols, packets and physical infrastructure — is essential for any IT professional.

Network topologies

  • Star — all devices connect to a central switch/hub. Most common in LANs. Single point of failure at the hub.
  • Bus — all devices share a single cable. Simple but collision-prone; obsolete in modern networks.
  • Ring — devices connected in a circular chain. Data travels in one direction. Used in some legacy token ring networks.
  • Mesh — every device connected to every other. Highly redundant; used in critical infrastructure and Wi-Fi mesh systems.
  • Hybrid — combinations of the above (most real networks).
Key Model

The OSI Model — 7 layers

Open Systems Interconnection model

The OSI model is a conceptual framework dividing network communication into 7 layers. Each layer has a specific function and communicates only with the layers directly above and below it.

# Layer Function Example
7ApplicationUser-facing servicesHTTP, FTP, SMTP, DNS
6PresentationData translation, encryptionSSL/TLS, JPEG, ASCII
5SessionOpens/closes connectionsNetBIOS, RPC
4TransportEnd-to-end deliveryTCP, UDP
3NetworkRouting between networksIP, ICMP, routers
2Data LinkNode-to-node transferEthernet, MAC, switches
1PhysicalElectrical signals, cablesCables, hubs, Wi-Fi signals

Protocols

TCP vs UDP

TCP — Transmission Control Protocol

Reliable, ordered, connection-based. Before sending data, TCP establishes a connection via a "three-way handshake" (SYN → SYN-ACK → ACK). Data is guaranteed to arrive in order; lost packets are retransmitted.

Used for: web browsing (HTTP/HTTPS), email, file transfers — where accuracy matters.

UDP — User Datagram Protocol

Fast, connectionless, unreliable. UDP sends packets without establishing a connection or guaranteeing delivery. If packets arrive out of order or are lost, they are not retransmitted.

Used for: video streaming, online gaming, VoIP, DNS — where speed matters more than perfection.
