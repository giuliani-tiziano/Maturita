Topic 09 · Networking & Internet

The Internet & the Web

ARPANET · TCP/IP · DNS · HTTP/HTTPS · Tim Berners-Lee · Web 1.0 / 2.0 / 3.0

History

From ARPANET to the World Wide Web

1969
ARPANET — the first internet
The US Department of Defense's Advanced Research Projects Agency creates ARPANET, connecting four universities (UCLA, Stanford, UCSB, Utah). The first message sent: "login" — the system crashed after "lo". ARPANET used packet switching: data is broken into packets, each routed independently.
1983
TCP/IP becomes the standard protocol
TCP/IP replaces older protocols, creating the technical foundation of the modern internet. All devices on the internet use TCP/IP to communicate.
1989–1991
Tim Berners-Lee invents the World Wide Web
At CERN, British scientist Tim Berners-Lee proposes a system of interlinked documents accessible via the internet — the World Wide Web. He creates HTTP (protocol), HTML (language), and the URL (address system). The first website goes live in 1991. The Web is NOT the internet — it is a service running on the internet.
2004–
Web 2.0 — the social web
Web 2.0 marks the shift from static pages to dynamic, user-generated content: Facebook (2004), YouTube (2005), Twitter (2006). Users are no longer just consumers but producers of content.
Key Protocols

How the web works — HTTP & DNS

HTTP & HTTPS

HTTP (HyperText Transfer Protocol) defines how web browsers and servers communicate. When you type a URL, your browser sends an HTTP request; the server responds with the page content.

HTTPS = HTTP + TLS encryption. The padlock icon in your browser confirms TLS is active: data between you and the server is encrypted. Essential for banking, e-commerce, login pages.

HTTP methods: GET (retrieve), POST (send data), PUT (update), DELETE.

DNS — the internet's phone book

The Domain Name System (DNS) translates human-readable domain names (google.com) into IP addresses (142.250.180.46) that computers use to find each other.

  1. You type "google.com" in your browser.
  2. Your computer asks a DNS resolver: "What is the IP of google.com?"
  3. The resolver queries DNS servers and returns the IP.
  4. Your browser connects to that IP — the Google server.
