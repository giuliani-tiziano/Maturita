Cloud Computing

IaaS · PaaS · SaaS · AWS · Azure · Virtualisation · Scalability

Overview

What is cloud computing?

Cloud computing is the delivery of computing services — servers, storage, databases, networking, software, analytics and intelligence — over the internet ("the cloud") on a pay-as-you-go basis. Instead of owning and maintaining physical data centres and servers, companies can rent access to anything from applications to storage from a cloud provider. Major providers: AWS (Amazon), Azure (Microsoft), Google Cloud.

Service Models

IaaS · PaaS · SaaS

The three service models

ModelFull nameExamples
IaaSInfrastructure as a ServiceAWS EC2, Azure VMs
PaaSPlatform as a ServiceHeroku, Google App Engine
SaaSSoftware as a ServiceGmail, Office 365, Salesforce

IaaS — you manage OS and applications; the provider manages hardware. PaaS — you manage only applications; the provider manages everything else. SaaS — you just use the software; the provider manages everything.

Deployment models

  • Public cloud — resources shared among many organisations; managed by provider (AWS, Azure). Most cost-effective; least control.
  • Private cloud — dedicated infrastructure for one organisation; on-premises or hosted. More control; higher cost.
  • Hybrid cloud — combination of public and private. Sensitive data on private; scalable workloads on public.
  • Multi-cloud — using services from multiple providers to avoid vendor lock-in.
Key Concepts

Virtualisation & scalability

Virtualisation

The technology that makes cloud computing possible. A hypervisor (e.g., VMware, KVM) runs on physical hardware and creates multiple Virtual Machines (VMs), each acting as an independent computer with its own OS. One physical server can run dozens of VMs simultaneously — maximising hardware utilisation.

Containers (Docker, Kubernetes) are lighter than VMs — they share the host OS kernel but isolate applications. Faster to start, more efficient.
