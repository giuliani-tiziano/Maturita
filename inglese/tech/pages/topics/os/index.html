Operating Systems

Kernel · Process Management · File Systems · Linux · Windows · macOS

Overview

What is an Operating System?

An Operating System (OS) is system software that manages hardware resources and provides services for application software. Without an OS, every application would need to directly control the hardware — an impossible task. The OS acts as an intermediary between users, applications and hardware. Major examples: Windows, Linux, macOS, Android, iOS.

Core OS functions

  • Process management — creates, schedules and terminates processes. Decides which process gets CPU time (scheduling algorithms: FIFO, Round Robin, Priority).
  • Memory management — allocates RAM to processes; uses virtual memory (paging) to extend available memory using disk space.
  • File management — organises data in files and directories; manages permissions (who can read/write/execute).
  • I/O management — controls input and output devices via drivers.
  • Security — user authentication, access control, process isolation.

The kernel

The kernel is the core of the OS — the part that directly interfaces with hardware. It runs in "kernel mode" with unrestricted access to hardware. Applications run in "user mode" with restricted access; they communicate with hardware only through the kernel (system calls).

Types of kernel architecture:

  • Monolithic — all OS services in one large kernel (Linux)
  • Microkernel — minimal kernel; services run in user space (minix)
  • Hybrid — mix of both (Windows NT, macOS XNU)
