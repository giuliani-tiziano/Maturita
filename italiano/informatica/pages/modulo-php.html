Informatica · Modulo 2

PHP
lato server

Il linguaggio server-side che dà vita alle applicazioni web dinamiche.

Architettura client/server

Il browser (client) invia una richiesta HTTP al server web. Il server esegue PHP, che elabora la logica, interroga MySQL, genera HTML dinamico e restituisce la risposta. Il client vede solo l'HTML — mai il codice PHP sorgente.

PHP vive sul server: ha accesso al filesystem, al database, alle sessioni. JavaScript vive nel browser: ha accesso al DOM, agli eventi, all'interfaccia utente.

phpMyAdmin

Interfaccia web grafica per gestire MySQL senza usare la riga di comando. Permette di: creare/eliminare database e tabelle, eseguire query SQL, importare/esportare dati (CSV, SQL), gestire utenti e permessi. Usato ampiamente in sviluppo; in produzione si preferisce un accesso più controllato.

Sezione 02

Sintassi base

Variabili, costanti, tipi, operatori, scope, superglobali, strutture di controllo.

Le variabili iniziano sempre con $ e sono case-sensitive: $nome e $Nome sono variabili diverse. PHP è a tipizzazione debole: il tipo è determinato automaticamente dal contesto.

Tipi principali: int, float, string, bool, array, null.

$nome = "Mario";           // string
$eta  = 20;               // int
$voto = 27.5;             // float
$prom = true;             // bool
$nulla = null;            // null

define('MAX_VOTO', 30);   // costante (maiuscolo per convenzione)
const MIN_VOTO = 18;      // alternativa moderna

Operatori: aritmetici (+ - * / % **), confronto (== === != !== < > <= >=), logici (&& || !), stringa (. concatenazione), ternario ($a ? $b : $c), Elvis ($a ?: $b).

In PHP le variabili hanno scope locale per default. Una variabile definita fuori da una funzione non è accessibile dentro (e viceversa). Per accedere a variabili globali dentro una funzione:

$contatore = 0;

function incrementa() {
    global $contatore;  // dichiara globale
    $contatore++;
}
// oppure: $GLOBALS['contatore']++;

Superglobali — accessibili ovunque senza dichiarare global:

Selezione:

if ($voto >= 60) {
    echo "Promosso";
} elseif ($voto >= 18) {
    echo "Ammesso";
} else {
    echo "Rimandato";
}

// Switch
switch ($giorno) {
    case "Lun": echo "Lunedì"; break;
    default: echo "Altro giorno";
}

// Operatore ternario
$msg = ($voto >= 60) ? "OK" : "No";

Iterazione:

// for — indice noto
for ($i = 0; $i < 10; $i++) { ... }

// while — condizione
while ($i > 0) { $i--; }

// do-while — almeno una volta
do { ... } while ($condizione);

// foreach — array (INDISPENSABILE)
foreach ($studenti as $nome) {
    echo $nome;
}
// con chiave:
foreach ($dati as $k => $v) {
    echo "$k: $v";
}
Sezione 03

Array e Funzioni

Array indicizzato (chiavi numeriche automatiche):

$voti = [28, 30, 25, 27];
echo $voti[0];  // 28
$voti[] = 29;   // aggiunge in coda

Array associativo (chiavi stringa):

$studente = [
    'nome'    => 'Mario',
    'cognome' => 'Rossi',
    'voto'    => 28
];
echo $studente['nome'];  // Mario

Funzioni principali: count() (dimensione), array_push/pop() (aggiunta/rimozione in coda), array_shift/unshift() (testa), sort()/rsort() (ordina valori), asort()/ksort() (mantiene chiavi), in_array() (cerca valore), array_key_exists() (cerca chiave), array_merge() (unisce), array_map() (trasforma), array_filter() (filtra), implode() (array→stringa), explode() (stringa→array).

// Funzione con parametri e valore di ritorno
function calcolaMedia(array $voti): float {
    return array_sum($voti) / count($voti);
}
echo calcolaMedia([28, 30, 25]);  // 27.67

// Parametri opzionali con default
function saluta(string $nome, string $titolo = "Sig."): string {
    return "Buongiorno, $titolo $nome!";
}

// Funzione anonima (lambda)
$doppio = function(int $n): int { return $n * 2; };
echo $doppio(5);  // 10

// Arrow function (PHP 7.4+)
$triplo = fn($n) => $n * 3;

// Passaggio per riferimento
function incrementa(int &$n): void { $n++; }
$x = 5;
incrementa($x);
echo $x;  // 6

Stringhe: strlen(), strtolower/strtoupper(), trim(), str_replace(), substr(), strpos(), sprintf() (formattazione).

Date: date('d/m/Y'), time() (timestamp), mktime(), strtotime().

Espressioni regolari (PCRE):

preg_match('/^\d{3}-\d{4}$/', $tel)  // verifica pattern
preg_replace('/[^a-z0-9]/i', '', $s) // sostituisce
preg_split('/,\s*/', $lista)          // divide stringa

Cookie (dati sul client):

// Imposta cookie (scade tra 7 giorni)
setcookie('tema', 'dark',
    time() + 7*24*3600, '/', '', true, true);
// path, secure, httpOnly

// Leggi cookie
$tema = $_COOKIE['tema'] ?? 'light';

// Elimina cookie
setcookie('tema', '', time()-1);

Sessione (dati sul server):

session_start(); // SEMPRE prima di tutto

// Salva dato di sessione
$_SESSION['user_id'] = 42;
$_SESSION['nome'] = 'Mario';

// Leggi
$id = $_SESSION['user_id'] ?? null;

// Distruggi al logout
session_destroy();
unset($_SESSION);

Differenza chiave

I cookie sono memorizzati nel browser e inviati ad ogni richiesta HTTP. Le sessioni memorizzano i dati sul server e inviano al browser solo un piccolo session_id (via cookie PHPSESSID). Per dati sensibili, le sessioni sono più sicure.

Mai salvare password in chiaro nel database. Usare sempre l'hashing:

// REGISTRAZIONE: hashing della password
$hash = password_hash($password_input, PASSWORD_BCRYPT);
// Salva $hash nel DB, non la password originale!

// LOGIN: verifica
$hash_dal_db = /* ... leggi dal DB ... */;
if (password_verify($password_input, $hash_dal_db)) {
    session_start();
    $_SESSION['user_id'] = $utente['id'];
    header('Location: dashboard.php');
    exit;
} else {
    $errore = "Credenziali errate";
}

// LOGOUT
session_start();
session_destroy();
header('Location: login.php');
exit;

Perché NON usare md5() o sha1()?

md5() e sha1() sono algoritmi veloci: un attaccante può provare miliardi di hash al secondo (brute force / rainbow table). password_hash() usa bcrypt che è volutamente lento e aggiunge un salt casuale automatico. Non è possibile "invertire" bcrypt.

Sezione 05

MySQLi e JSON

// Connessione
$conn = new mysqli('localhost', 'utente', 'password', 'db_nome');
if ($conn->connect_error) {
    die("Connessione fallita: " . $conn->connect_error);
}
$conn->set_charset('utf8mb4');

// Query semplice (solo dati trusted, es. senza input utente)
$result = $conn->query("SELECT * FROM studenti ORDER BY cognome");
while ($riga = $result->fetch_assoc()) {
    echo $riga['nome'] . " " . $riga['cognome'];
}

// Chiusura
$conn->close();

Quando si usano dati provenienti dall'utente ($_POST, $_GET), si devono SEMPRE usare i prepared statements:

// SELECT con parametri
$stmt = $conn->prepare("SELECT * FROM utenti WHERE email = ? AND attivo = ?");
$stmt->bind_param("si", $email, $attivo); // s=string, i=int
$stmt->execute();
$result = $stmt->get_result();
$utente = $result->fetch_assoc();
$stmt->close();

// INSERT con parametri
$stmt = $conn->prepare("INSERT INTO studenti (nome, voto) VALUES (?, ?)");
$stmt->bind_param("si", $nome, $voto);
$stmt->execute();
$id_inserito = $conn->insert_id; // ID dell'ultima riga inserita
$stmt->close();

Perché i prepared statements prevengono SQL injection?

Il driver MySQL separa la struttura della query (?) dai parametri. I parametri vengono inviati al server dopo la compilazione della query: il server sa già che sono dati, non istruzioni SQL. Un input come ' OR '1'='1 viene trattato come semplice stringa, non come codice.

JSON (JavaScript Object Notation) è il formato standard per scambiare dati tra PHP e JavaScript tramite AJAX:

// PHP → JSON (risposta per richiesta AJAX)
$dati = [
    'studenti' => [
        ['nome' => 'Mario', 'voto' => 28],
        ['nome' => 'Luigi', 'voto' => 30]
    ],
    'totale' => 2
];

header('Content-Type: application/json');
echo json_encode($dati);
exit;

// JSON → PHP (se ricevi JSON in input)
$body = file_get_contents('php://input');
$obj  = json_decode($body, true); // true = array associativo
echo $obj['nome'];  // Mario
